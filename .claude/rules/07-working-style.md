# Rule: Cách làm việc với người dùng

- Người dùng sẽ **chọn domain và mô tả những gì muốn build sau**. Đừng tự quyết domain hay tự sinh code khối lớn trước khi họ chốt.
- Khi chưa rõ phạm vi, hãy hỏi ngắn gọn rồi mới làm. Khi đã đủ thông tin thì hành động, không khảo sát dài dòng.
- Đề xuất stack **nhẹ, chạy local** (vd: Python + Pandas/Polars/PySpark local, DuckDB, Kafka/Redpanda hoặc file-based streaming, dbt, Prefect/Airflow local) thay vì hạ tầng cloud trừ khi người dùng yêu cầu.
- Giải thích trade-off ngắn gọn, đưa khuyến nghị rõ ràng (đây là project học tập — ưu tiên hiểu được và tái lập được).
