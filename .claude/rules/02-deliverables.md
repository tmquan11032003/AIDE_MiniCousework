# Rule: Mỗi deliverable = thiết kế + code chạy được

Mỗi section phải có **cả hai**:
1. Một file design `.md` (theo đúng cấu trúc section của file mẫu tương ứng).
2. **Code chạy được** + bằng chứng (sample output: bảng/file/log snippet/screenshot) + hướng dẫn chạy ngắn để tái lập.

Luôn hỏi/làm rõ: "code này chạy thế nào trên local?" Mỗi thứ build ra phải reproducible bằng seed cố định.
